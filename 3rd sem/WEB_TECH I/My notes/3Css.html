3.1 Need of CSS

CSS (Cascading Style Sheets) is used to style HTML elements. It separates content (HTML) from design, making web pages visually appealing and easier to maintain.

3.2 CSS Basics

3.2.1.1 CSS Syntax

CSS syntax consists of a selector and a declaration block. The selector specifies the HTML element to style, and the declaration block contains style rules.

  • Selector: The HTML element to style.
  • Declaration Block: Contains style rules in key-value pairs.
Example: 

    /* CSS Syntax */
    h1 {
        color: blue;
        font-size: 24px;
    }

3.2.1.1 selectors

CSS selectors are used to target specific HTML elements for styling. There are different types of selectors like element, class, id, and pseudo-class selectors.

Selector Type Description Example
Element Selector Selects elements based on their tag name. h1 { color: blue; }
Class Selector Selects elements based on their class attribute. .highlight { background-color: yellow; }
ID Selector Selects elements based on their id attribute. #header { font-size: 20px; }
Pseudo-class Selector Selects elements based on their state or position. a:hover { text-decoration: underline; }

Declarations:

Declarations are key-value pairs that define the style of an element. Each declaration consists of a property and a value separated by a colon.

  • Property: The CSS attribute to style.
  • Value: The value assigned to the property.
Example: 

    /* Declaration Block */
    h1 {
        color: blue;
        font-size: 24px;
    }

Types of CSS:

CSS can be applied in three ways: inline, internal, and external. Inline CSS is applied directly to an HTML element, internal CSS is defined in the head section of an HTML document, and external CSS is stored in a separate file and linked to the HTML document.

3.2.1.2 Types of CSS

CSS can be applied in three ways: inline, internal, and external. Each method has its own use case and advantages.

Type Description Example Pros Cons
Inline CSS Applied directly to an HTML element using the style attribute. <h1 style="color: blue;">Hello World</h1>
  • Quick to apply
  • Overrides other styles
  • Not reusable
  • Harder to maintain
Internal CSS Defined in the head section of an HTML document using the <style> tag. 

<head>
    <style>
        h1 {
            color: blue;
        }
    </style>
</head>
  • Useful for single-page styles
  • Overrides external styles
  • Less maintainable than external CSS
  • Not reusable across multiple pages
External CSS Stored in a separate file with a .css extension and linked to the HTML document using the <link> tag. 

<head>
    <link rel="stylesheet" href="styles.css">
</head>
  • Most maintainable
  • Reusable across multiple pages
  • Requires additional HTTP request
  • May be overridden by inline or internal styles

Ranking:

The ranking of CSS types in terms of specificity and maintainability:

  1. External CSS: Most maintainable and reusable, as styles are defined in a separate file.
  2. Internal CSS: Useful for single-page styles, but less maintainable than external CSS.
  3. Inline CSS: Least maintainable, as styles are applied directly to elements, making it harder to manage.

Grouping Selectors

Grouping selectors allows applying the same styles to multiple elements. This helps to reduce redundancy and keep the CSS code clean and efficient.


/* Grouping Selectors */
h1, h2, h3 {
    color: blue;
}

Pseudo-classes & Pseudo-elements:

Pseudo-classes and pseudo-elements are used to style elements based on their state or position. They provide additional styling options beyond standard selectors.

  • Pseudo-classes: Select elements based on user interaction or state.
  • Pseudo-elements: Style specific parts of an element.
Example: 

/* Pseudo-classes */
a:hover {
    text-decoration: underline;
}

/* Pseudo-elements */

p::first-line {
    font-weight: bold;
}

comments

Comments in CSS are used to add notes or descriptions to the code. They are ignored by the browser and are useful for documenting styles or temporarily disabling code.


/* This is a CSS comment */

3.2.2 CSS Specificity

Selector Type Specificity Syntax
Inline Styles 1000 <h1 style="color: blue;">Hello</h1>
ID Selectors 0100 #header { color: blue; }
Class/Attribute Selectors 0010 .highlight { color: blue; }
Element/Type Selectors 0001 h1 { color: blue; }
Universal Selector 0000 * { color: blue; }

Specificity is calculated based on the number of occurrences of each selector type. The specificity value is a four-digit number, with each digit representing a selector type.

3.2.3 Internal vs External CSS

Criteria Internal CSS External CSS
Definition Defined within the <style> tag in the head section of an HTML document. Stored in a separate file with a .css extension and linked using the <link> tag.
Usage Useful for single-page styles or when specific styles are needed for a single document. Best for applying consistent styles across multiple pages.
Maintainability Less maintainable as styles are embedded within the HTML document. More maintainable as styles are separated from the HTML document.
Reusability Not reusable across multiple pages. Reusable across multiple pages.
Performance May increase the size of the HTML document, potentially affecting load times. Requires an additional HTTP request but keeps HTML document size smaller.
Specificity Overrides external styles but can be overridden by inline styles. Can be overridden by internal and inline styles.

3.2.4 Style Text Fonts, Color, Links

Category Description Example
Fonts Use font-family, font-size, and font-weight to style text fonts. 

/* Font Family */
p {
    font-family: Arial, sans-serif;
}

/* Font Size */
h1 {
    font-size: 24px;
}

/* Font Weight */
strong {
    font-weight: bold;
}
Colors Use color and background-color with names, hex, or RGB values to style text colors. 

/* Text Color */
p {
    color: blue;
}

/* Background Color */
div {
    background-color: #f0f0f0;
}

/* RGB Color */
h1 {
    color: rgb(255, 0, 0);
}
Links Style links using :hover, :visited, and :active pseudo-classes. 

/* Link Hover */
a:hover {
    text-decoration: underline;
}

/* Visited Link */
a:visited {
    color: purple;
}

/* Active Link */
a:active {
    color: red;
}

3.3 CSS Box Model

The CSS box model defines the content, padding, border, and margin of an element. Understanding the box model is essential for creating layouts and spacing elements on a web page.

3.3.1 Box Model Properties

Property Description Example
Width Sets the width of an element's content area. width: 200px;
Height Sets the height of an element's content area. height: 100px;
Padding Space between the content and the border. padding: 10px;
Border Border around the padding and content. border: 1px solid black;
Margin Space outside the border. margin: 20px;

3.3.2 Box Model Example

Example Box
div { width: 150px; padding: 20px; border: 5px solid black; margin: 10px; background: #f0f0f0; }

3.4 CSS Layouts

3.4.1 Flexbox Layout

Flexbox is a layout model that allows elements to align and distribute space within a container. It provides a more efficient way to create responsive layouts compared to traditional methods.

Flexbox Properties:

  • display: Defines a flex container.
  • flex-direction: Specifies the direction of the flex container.
  • justify-content: Aligns items along the main axis.
  • align-items: Aligns items along the cross axis.
  • flex: Specifies the flexibility of a flex item.

3.4.2 Grid Layout

Grid layout is a two-dimensional layout system that allows elements to be placed in rows and columns. It provides a powerful way to create complex layouts with precise control over the placement of elements.

Grid Properties:

  • display: Defines a grid container.
  • grid-template-columns: Specifies the size of grid columns.
  • grid-template-rows: Specifies the size of grid rows.
  • grid-gap: Sets the gap between grid items.
  • grid-column: Specifies the grid columns a grid item spans.

3.4.3 Responsive Design

Responsive design is an approach to web design that makes web pages render well on a variety of devices and window or screen sizes. It uses a combination of flexible grids, layouts, images, and CSS media queries to create a responsive user experience.

Media Queries:

Media queries are used to apply different styles based on the device's characteristics, such as screen width, height, orientation, and resolution. They allow developers to create responsive designs that adapt to different devices and screen sizes.


/* Media Query Example */
@media screen and (max-width: 600px) {
    body {
        font-size: 14px;
    }
}

3.4.4 CSS Frameworks

CSS frameworks are pre-designed libraries that provide a set of styles, components, and layouts to help developers build responsive websites quickly. Popular CSS frameworks include Bootstrap, Foundation, and Bulma.

Benefits of CSS Frameworks:

  • Consistent design across multiple pages.
  • Responsive layouts out of the box.
  • Reusable components like buttons, forms, and navigation bars.

3.4.5 CSS Preprocessors

CSS preprocessors are tools that extend the functionality of CSS by adding features like variables, nesting, and mixins. They help developers write cleaner, more maintainable CSS code and improve the efficiency of the styling process.

Popular CSS Preprocessors:

  • Sass: Syntactically Awesome Stylesheets.
  • Less: Leaner Style Sheets.
  • Stylus: Expressive, dynamic, and robust CSS.

3.6 CSS Preprocessors (SASS, LESS)

3.6.1 SASS

Sass (Syntactically Awesome Stylesheets) is a CSS preprocessor that extends the functionality of CSS with features like variables, nesting, and mixins. It helps developers write cleaner, more maintainable CSS code and improve the efficiency of the styling process.

Sass Features:

  • Variables: Define reusable values for colors, fonts, and other properties.
  • Nesting: Organize CSS rules in a hierarchical structure for better readability.
  • Mixins: Create reusable blocks of styles that can be included in multiple rules.
  • Functions: Define custom functions to generate dynamic styles based on input values.

3.6.2 LESS

Less (Leaner Style Sheets) is a CSS preprocessor that extends the functionality of CSS with features like variables, mixins, and functions. It helps developers write cleaner, more maintainable CSS code and improve the efficiency of the styling process.

Less Features:

  • Variables: Define reusable values for colors, fonts, and other properties.
  • Mixins: Create reusable blocks of styles that can be included in multiple rules.
  • Functions: Define custom functions to generate dynamic styles based on input values.
  • Nesting: Organize CSS rules in a hierarchical structure for better readability.
